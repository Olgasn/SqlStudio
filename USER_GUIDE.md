# Руководство пользователя SlqStudio

## 1. Назначение системы

`SlqStudio` представляет собой учебное веб-приложение, предназначенное для практической отработки навыков работы с языком SQL.

С помощью данной системы пользователь имеет возможность выполнить вход через систему Moodle с указанием адреса электронной почты и выбранного курса, получить индивидуальный вариант заданий, составлять и проверять SQL-решения, отслеживать состояние выполнения заданий, а также формировать итоговый отчёт по выполненным работам.

Пользователь, обладающий ролью `EditingTeacher`, дополнительно получает доступ к административным функциям системы.

## 2. Вход в систему

### 2.1. Открытие страницы входа

Для начала работы откройте страницу входа, расположенную по адресу `/Auth/Login`.

### 2.2. Авторизация

На странице входа необходимо выполнить следующие действия:

1. Введите адрес электронной почты.
2. Выберите курс из предложенного списка.
3. Нажмите кнопку `Войти`.

В случае успешной авторизации система создаёт пользовательскую сессию и открывает главную страницу приложения.

### 2.3. Выход из системы

Для завершения работы нажмите кнопку `Выйти`, расположенную на верхней панели интерфейса.

## 3. Навигация

После входа в систему основная навигация осуществляется посредством бокового меню.

Всем пользователям доступны следующие разделы:

- раздел `Изучение SQL`, содержащий перечень лабораторных работ;
- раздел `О проекте (GitHub)`, представляющий собой ссылку на репозиторий проекта.

Пользователю с ролью `EditingTeacher` дополнительно доступны создание лабораторных работ и просмотр их перечня, создание заданий и просмотр их перечня, редактирование конфигурации, а также просмотр журналов.

## 4. Порядок работы студента

### 4.1. Выбор лабораторной работы

1. Перейдите в раздел `Изучение SQL`.
2. Выберите в таблице необходимую лабораторную работу.
3. Нажмите кнопку просмотра заданий, обозначенную соответствующим значком.

В результате выполнения указанных действий открывается страница `Ход выполнения работ`.

### 4.2. Выполнение задания

1. На странице `Ход выполнения работ` нажмите на строку требуемого задания.
2. После этого открывается редактор SQL.
3. При необходимости откройте диаграмму базы данных с помощью кнопки `Показать диаграмму БД`.
4. Введите текст SQL-запроса и нажмите кнопку `Запустить`.

При выполнении запроса система последовательно осуществляет проверку синтаксиса SQL, сравнение полученного результата с ожидаемым, а также сохранение состояния задания в текущей сессии.

### 4.3. Состояния заданий

На странице `Ход выполнения работ` для обозначения состояния заданий применяется цветовая индикация:

- состояние `Выполнено` отображается зелёным цветом;
- состояние `Ошибка` отображается красным цветом;
- состояние `Не начато` соответствует заданию, выполнение которого ещё не запускалось.

### 4.4. Завершение работы и формирование отчёта

1. На странице `Ход выполнения работ` нажмите кнопку `Закончить`.
2. После этого открывается страница отчёта.
3. Нажмите кнопку `Отправить отчет`.

В результате система формирует отчёт в формате PDF, предоставляет данный документ для скачивания в браузере, а также выполняет его отправку по электронной почте в соответствии с серверными настройками.

## 5. Порядок работы преподавателя и администратора (роль `EditingTeacher`)

### 5.1. Лабораторные работы

Управление лабораторными работами осуществляется в разделах `Создать лабораторную работу` и `Список лабораторных работ`. Преподаватель имеет возможность создавать, редактировать, удалять и просматривать лабораторные работы.

### 5.2. Задания

Управление заданиями осуществляется в разделах `Создать задание` и `Список заданий`. Преподаватель имеет возможность создавать, редактировать, удалять и просматривать задания.

### 5.3. Просмотр журналов

В разделе `Просмотр логов` пользователю предоставляется возможность просматривать перечень файлов журналов, открывать содержимое отдельного файла, скачивать его, а также удалять.

### 5.4. Редактирование конфигурации и управление курсами

Страница конфигурации доступна по адресу `/Config/Login`. Она использует отдельную авторизацию по учётным данным секции `ConfigUser` и **не зависит** от Moodle и JWT, поэтому доступна даже до создания первого курса.

После входа открывается раздел настроек, в котором доступны:

- **Управление курсами** — создание и удаление курсов. Это необходимо при первом развёртывании и при добавлении новых курсов Moodle. Название курса должно точно совпадать с названием курса в Moodle.
- **Редактирование конфигурации** — изменение параметров файла `appsettings.json` через веб-интерфейс: строки подключения к базам данных, параметры Moodle, SMTP и другие. Поля снабжены поясняющими комментариями. Чтобы сохранить изменения, нажмите кнопку `Сохранить`.

> **Важно:** Страница конфигурации позволяет изменить любые параметры приложения. Ограничьте к ней доступ на уровне сети или обратного прокси-сервера.

## 6. Сообщения системы и типовые ситуации

### 6.1. Сообщение `Доступ запрещён`

Данное сообщение возникает в том случае, когда у пользователя недостаточно прав для доступа к выбранному разделу. Для устранения ситуации следует выполнить вход под учётной записью с необходимой ролью либо обратиться к администратору системы Moodle.

### 6.2. Невозможность входа на странице `/Auth/Login`

Невозможность входа может быть обусловлена одной из следующих причин:

- Пользователь не найден в Moodle по указанному адресу электронной почты.
- Курс не выбран или список курсов пуст.
- Выбранный курс отсутствует в базе данных приложения — его необходимо добавить через `/Config/Login` → **Управление курсами**.
- Роль пользователя в выбранном курсе не удалось определить.

**При первом развёртывании** список курсов на странице входа всегда пуст — необходимо сначала создать хотя бы один курс через `/Config/Login`.

### 6.3. Ошибка SQL в редакторе

Ошибка при выполнении SQL-запроса может быть вызвана синтаксической ошибкой, несоответствием типа SQL-команды ожидаемому для данного задания типу либо логической ошибкой запроса, при которой полученный результат не совпадает с эталонным. Для устранения ошибки следует исправить запрос и повторно нажать кнопку `Запустить`.

### 6.4. Завершение сессии

При длительном отсутствии действий пользователя сессия может быть завершена по истечении установленного времени. В этом случае необходимо повторно выполнить вход в систему.

## 7. Первоначальный запуск системы

При первом развёртывании база данных пуста и войти через `/Auth/Login` невозможно. Выполните следующие шаги:

1. Откройте `/Config/Login`.
2. Введите учётные данные администратора (`ConfigUser.Name` и `ConfigUser.Password` из конфигурации).
3. Нажмите кнопку **Управление курсами**.
4. Нажмите **Добавить курс**, введите название (должно точно совпадать с названием курса в Moodle) и сохраните.
5. Вернитесь на `/Auth/Login` — список курсов появится, вход в систему станет доступен.

После входа преподаватель с ролью `editingteacher` может создавать лабораторные работы и задания через административный интерфейс.

## 8. Перечень основных адресов

- Страница входа: `/Auth/Login`.
- Главная страница: `/Home/Index`.
- Лабораторные работы: `/LabWorks/LabList`.
- Отчёт: `/Report/Index`.
- Журналы (для роли `EditingTeacher`): `/Logs/Index`.
- Конфигурация (отдельная авторизация по `ConfigUser`): `/Config/Login`.
- Управление курсами: `/Config/CoursesList`.
- Редактирование параметров: `/Config/EditConfig`.
