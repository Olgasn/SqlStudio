# Руководство пользователя SlqStudio

## 1. Назначение системы

`SlqStudio` — учебное веб-приложение для практики SQL.

С помощью системы пользователь может:

- войти через Moodle (email + курс),
- получить индивидуальный вариант заданий,
- писать SQL-решения и проверять их,
- видеть статус выполнения задач,
- сформировать итоговый отчет по выполненным работам.

Пользователь с ролью `EditingTeacher` дополнительно получает доступ к административным функциям.

## 2. Вход в систему

### 2.1. Открытие страницы входа

Откройте страницу:

- `/Auth/Login`

### 2.2. Авторизация

На странице входа:

1. Введите email.
2. Выберите курс из списка.
3. Нажмите `Войти`.

Если авторизация успешна, система создаст сессию и откроет главную страницу.

### 2.3. Выход

Для выхода нажмите кнопку `Выйти` в верхней панели.

## 3. Навигация

После входа основная навигация доступна через боковое меню.

Основные разделы:

- `Изучение SQL` — список лабораторных работ.
- `О проекте (GitHub)` — ссылка на репозиторий.

Для роли `EditingTeacher` дополнительно доступны:

- создание и список лабораторных работ,
- создание и список заданий,
- редактирование конфигурации,
- просмотр логов.

## 4. Сценарий пользователя (студент)

### 4.1. Выбор лабораторной работы

1. Перейдите в `Изучение SQL`.
2. В таблице выберите лабораторную работу.
3. Нажмите на кнопку просмотра заданий (иконка задания).

Откроется страница `Ход выполнения работ`.

### 4.2. Выполнение задания

1. На странице `Ход выполнения работ` нажмите на строку нужного задания.
2. Откроется SQL-редактор.
3. При необходимости откройте диаграмму БД кнопкой `Показать диаграмму БД`.
4. Введите SQL и нажмите `Запустить`.

Система выполняет:

- проверку синтаксиса SQL,
- сравнение результата с ожидаемым,
- сохранение статуса задачи в текущей сессии.

### 4.3. Статусы задач

На странице `Ход выполнения работ` используется цветовая индикация:

- `Выполнено` — зеленый статус,
- `Ошибка` — красный статус,
- `Не начато` — задача еще не запускалась.

### 4.4. Завершение работы и отчет

1. На странице `Ход выполнения работ` нажмите `Закончить`.
2. Откроется страница отчета.
3. Нажмите `Отправить отчет`.

Результат:

- формируется PDF-отчет,
- PDF отдается на скачивание в браузере,
- выполняется отправка email по серверным настройкам.

## 5. Сценарий преподавателя/администратора (`EditingTeacher`)

### 5.1. Лабораторные работы

Разделы:

- `Создать лабораторную работу`
- `Список лабораторных работ`

Доступные действия:

- создание,
- редактирование,
- удаление,
- просмотр.

### 5.2. Задания

Разделы:

- `Создать задание`
- `Список заданий`

Доступные действия:

- создание,
- редактирование,
- удаление,
- просмотр.

### 5.3. Просмотр логов

Раздел: `Просмотр логов`.

Возможности:

- просмотр списка лог-файлов,
- открытие содержимого файла,
- скачивание,
- удаление.

### 5.4. Редактирование конфигурации

Раздел: `Настройки конфигурации`.

Важно:

- для этого раздела используется отдельный вход по `ConfigUser` (из `appsettings.json`),
- после входа доступны поля конфигурации с комментариями,
- после изменения параметров нажмите `Сохранить`.

## 6. Сообщения и типовые ситуации

### 6.1. `Доступ запрещен`

Причина:

- у пользователя недостаточно прав для раздела.

Решение:

- войти под пользователем с нужной ролью,
- обратиться к администратору Moodle.

### 6.2. Не удается войти на странице `/Auth/Login`

Возможные причины:

- пользователь не найден в Moodle,
- не выбран курс,
- выбранный курс отсутствует в БД приложения,
- не удалось определить роль пользователя в курсе.

### 6.3. Ошибка SQL в редакторе

Причина:

- синтаксическая ошибка,
- тип SQL-команды не совпадает с ожидаемым для задания,
- логическая ошибка запроса (результат не совпадает с эталоном).

Решение:

- исправить запрос и повторно нажать `Запустить`.

### 6.4. Сессия завершилась

Если долго не выполнять действий, сессия может истечь.  
В этом случае выполните вход снова.

## 7. Полезные адреса

- Вход: `/Auth/Login`
- Главная: `/Home/Index`
- Лабораторные работы: `/LabWorks/LabList`
- Отчет: `/Report/Index`
- Логи (для `EditingTeacher`): `/Logs/Index`
- Конфигурация (для `EditingTeacher` + `ConfigUser`): `/Config/EditConfig`
