# Настройка интеграции с Moodle

Данный документ предназначен для администратора системы Moodle и описывает действия, необходимые для подключения SlqStudio к Moodle.

## Функции REST API, используемые приложением

SlqStudio обращается к следующим функциям REST API системы Moodle:

- `core_user_get_users_by_field` — поиск пользователя по адресу электронной почты при входе в систему;
- `core_course_get_courses` — получение списка курсов и поиск выбранного курса;
- `core_user_get_course_user_profiles` — определение роли пользователя в выбранном курсе.

## Настройка на стороне Moodle

Выполните следующие действия в административном разделе Moodle:

1. Включите подсистему веб-сервисов: **Администрирование сайта → Плагины → Веб-сервисы → Обзор**.
2. Активируйте протокол REST: **Администрирование сайта → Плагины → Веб-сервисы → Управление протоколами**.
3. Создайте внешний сервис: **Администрирование сайта → Плагины → Веб-сервисы → Внешние сервисы → Добавить новый сервис**.
4. Добавьте в состав созданного сервиса три функции:
   - `core_user_get_users_by_field`;
   - `core_course_get_courses`;
   - `core_user_get_course_user_profiles`.
5. Создайте служебную учётную запись Moodle или выберите существующую, назначьте её созданному сервису.
6. Сформируйте токен доступа для этой учётной записи: **Администрирование сайта → Плагины → Веб-сервисы → Управление токенами**.
7. Убедитесь, что участникам курсов назначены роли в Moodle. Для доступа к административным функциям SlqStudio требуется роль `editingteacher`.

## Настройка на стороне SlqStudio

Внесите полученные данные в файл `SlqStudio.Web/appsettings.json`:

```json
"MoodleApi": {
  "MoodleUrl": "https://moodle.example.com/webservice/rest/server.php",
  "Token": "your-moodle-webservice-token"
}
```

При настройке необходимо учитывать следующее:

- Значение `MoodleUrl` должно указывать на адрес `.../webservice/rest/server.php`.
- При входе пользователь выбирает курс из локальной базы данных SlqStudio; система затем ищет этот курс в Moodle по полю `displayname`.
- Значение `Course.Name` в локальной базе данных должно точно совпадать с наименованием курса в Moodle.
- Права пользователя в SlqStudio определяются по краткому наименованию (`ShortName`) его роли в Moodle; для административных операций требуется роль `editingteacher`.

В промышленной среде передавайте параметры через переменные окружения:

```powershell
$env:MoodleApi__MoodleUrl="https://moodle.example.com/webservice/rest/server.php"
$env:MoodleApi__Token="your-moodle-webservice-token"
```

## Проверка подключения

Чтобы убедиться в корректности токена и доступности API, выполните следующий запрос:

```powershell
$url = "https://moodle.example.com/webservice/rest/server.php?wstoken=YOUR_TOKEN&wsfunction=core_course_get_courses&moodlewsrestformat=json"
Invoke-RestMethod -Method Get -Uri $url
```

Если ответ содержит список курсов в формате JSON, подключение настроено правильно.

## Возможные ошибки и их причины

| Сообщение | Вероятная причина |
|---|---|
| `invalidtoken` | Токен указан неверно или срок его действия истёк. |
| `accessexception` / `Access control exception` | В состав внешнего сервиса не добавлены необходимые функции, либо служебная учётная запись не обладает достаточными правами. |
| «Пользователь с таким email не найден» | Указанный адрес электронной почты отсутствует в Moodle или введён с ошибкой. |
| «Выбранный курс не найден» | Наименование курса в SlqStudio не совпадает со значением `displayname` в Moodle. |
| «Не удалось определить вашу роль» | Пользователь не имеет назначенной роли в выбранном курсе Moodle. |
